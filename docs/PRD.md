# PRD: YouTube 테크 리뷰 트렌드 분석기 (TechPulse)

## 1. 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | TechPulse — YouTube 테크 리뷰 트렌드 분석기 |
| 버전 | v1.0 |
| 작성일 | 2026-03-20 |
| 대상 사용자 | 테크 마케터, 기업 시장조사팀, 테크 유튜버/크리에이터 |
| 핵심 가치 | "최근 테크 리뷰 트렌드를 한눈에 파악하고, 다음 콘텐츠/마케팅 전략을 데이터 기반으로 결정한다" |

---

## 2. 문제 정의

### 2.1 현재 상황
- 테크 시장 동향 파악을 위해 YouTube를 수동으로 검색·분석해야 함
- 키워드 트렌드 변화를 시계열로 추적할 방법이 없음
- 팀원 간 인사이트 공유가 비체계적 (스크린샷, 구두 전달)
- 글로벌 vs 국내 트렌드 비교가 어려움

### 2.2 목표 상태
- 자동 수집 → 분석 → 대시보드 시각화 → 리포트 발송까지 원스톱
- 주간/월간 트렌드 비교로 시장 흐름의 변곡점을 조기 포착
- 팀 전체가 하나의 대시보드에서 동일한 데이터로 의사결정

|---------|------|--------|------|

| P0 | 글로벌(영어) 지원 | Phase 1 | 한국어 + 영어 동시 검색, 언어별 결과 분리/통합 |

### 4.2 Phase 1: 핵심 분석 + 대시보드 (MVP)

#### 4.2.1 데이터 수집 엔진 (Backend — Python)
```
기능:
  - YouTube Data API v3로 테크 리뷰 영상 검색
  - 검색어: 한국어 6개 + 영어 6개 (설정 가능)
  - 수집 항목: 제목, 채널명, 조회수, 좋아요, 태그, 설명, 게시일
  - 최근 7일 / 30일 선택 가능
  - 중복 영상 제거 (video ID 기준)
```

#### 4.2.2 키워드 분석 엔진 (Backend — Python)
```
기능:
  - 제목 + 태그 + 설명에서 키워드 추출
  - 한/영 불용어 필터링
  - 빈도 기반 키워드 랭킹 TOP 10
  - 8개 관심사 카테고리 자동 분류
    (스마트폰, 노트북/PC, AI/소프트웨어, 오디오/웨어러블,
     카메라/영상, 성능/벤치마크, 가격/가성비, 디스플레이/디자인)
  - 언어별(한/영) 결과 분리 및 통합 뷰
```

#### 4.2.3 REST API (Backend — Python FastAPI)
```
엔드포인트:
  GET  /api/analyze          — 실시간 분석 실행 및 결과 반환
  GET  /api/keywords         — 키워드 TOP N 조회
  GET  /api/interests        — 관심사 카테고리 점수 조회
  GET  /api/videos           — 조회수 상위 영상 목록
  GET  /api/trends           — 주간/월간 트렌드 비교 (Phase 2)
  GET  /api/export/csv       — CSV 다운로드
  GET  /api/export/xlsx      — Excel 다운로드
```

#### 4.2.4 웹 대시보드 (Frontend — Next.js + React)
```
페이지:
  /                  — 대시보드 메인 (요약 카드 + 차트)
  /keywords          — 키워드 상세 분석
  /videos            — 상위 영상 리스트
  /trends            — 트렌드 변화 그래프 (Phase 2)
  /settings          — 검색어 설정, API 키 관리
  /login             — 인증 (Phase 3)

컴포넌트:
  - KeywordRankingChart: 바 차트 (키워드 빈도)
  - InterestRadarChart: 레이더 차트 (관심사 분포)
  - TrendLineChart: 라인 차트 (주간 트렌드 변화, Phase 2)
  - VideoTable: 상위 영상 테이블 (썸네일, 제목, 채널, 조회수)
  - ExportButton: CSV/Excel 다운로드 버튼
  - RegionToggle: 한국/글로벌/통합 전환
```

### 4.3 Phase 2: 트렌드 추적 + 자동화

#### 4.3.1 데이터 저장 (클라우드 DB)
```
DB: Supabase (PostgreSQL)
테이블:
  analysis_runs     — 분석 실행 이력 (id, run_date, region, status)
  videos            — 수집된 영상 (video_id, title, channel, views, likes, published_at, ...)
  keywords          — 키워드 집계 (run_id, keyword, count, category, language)
  trend_snapshots   — 주간 스냅샷 (week, keyword, rank, count, rank_change)
```

#### 4.3.2 트렌드 비교 로직
```
기능:
  - 이번 주 vs 지난 주 키워드 순위 변동 (▲▼)
  - 신규 진입 키워드 하이라이트
  - 월간 TOP 키워드 히트맵
  - 카테고리별 관심도 추이 그래프
```

#### 4.3.3 스케줄링
```
방식: Vercel Cron Jobs
주기:
  - 매일 06:00 KST — 데이터 수집 + 분석
  - 매주 월요일 09:00 KST — 주간 리포트 생성
```

---

### 4.4 Phase 3: 리포트 + 팀 협업

#### 4.4.1 이메일 리포트
```
발송: Resend API 또는 SendGrid
내용:
  - 주간 키워드 TOP 10 (순위 변동 포함)
  - 주요 관심사 분포 차트 (이미지 첨부)
  - 조회수 1위 영상 하이라이트
  - 대시보드 바로가기 링크
수신자: 팀 설정 페이지에서 관리
```

#### 4.4.2 팀 인증/멀티유저
```
인증: NextAuth.js (Google OAuth + 이메일 로그인)
권한:
  - Admin: 설정 변경, 팀원 초대, API 키 관리
  - Member: 대시보드 조회, 리포트 수신, 내보내기
```

## 5. 기술 스택

| 레이어 | 기술 | 이유 |
|--------|------|------|
| Frontend | Next.js 14 (App Router), React 18, TypeScript | 본격 웹앱 요구사항, SSR/ISR 지원 |

| 분석 엔진 | Python 3.11+ | 기존 analyzer.py 자산 활용, NLP 생태계 |
| DB | Supabase (PostgreSQL) | 무료 티어, 실시간 기능, Auth 내장 |
| 인증 | NextAuth.js + Supabase Auth | Google OAuth, 팀 관리 |
| 배포 | Vercel (Frontend) + Vercel Serverless (API) | 사용자 요구사항, CI/CD 내장 |
| 스케줄링 | Vercel Cron Jobs | 인프라 단순화 |
| 이메일 | Resend | Next.js 친화적, React Email 지원 |
| 내보내기 | SheetJS (xlsx), json2csv | 브라우저/서버 양쪽 지원 |

## 7. 데이터 모델

### 7.1 analysis_runs
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| run_date | TIMESTAMP | 실행 시각 |
| region | ENUM('kr', 'global', 'all') | 분석 지역 |
| period_days | INT | 분석 기간 (7, 30) |
| video_count | INT | 수집 영상 수 |
| status | ENUM('running', 'completed', 'failed') | 상태 |

### 7.2 videos
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| run_id | UUID | FK → analysis_runs |
| video_id | VARCHAR | YouTube video ID |
| title | TEXT | 영상 제목 |
| channel | VARCHAR | 채널명 |
| views | BIGINT | 조회수 |
| likes | BIGINT | 좋아요 |
| tags | TEXT[] | 태그 배열 |
| published_at | TIMESTAMP | 게시일 |
| language | VARCHAR | 감지된 언어 |

### 7.3 keywords
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| run_id | UUID | FK → analysis_runs |
| keyword | VARCHAR | 키워드 |
| count | INT | 출현 빈도 |
| category | VARCHAR | 관심사 카테고리 |
| language | VARCHAR | 언어 (ko/en) |

### 7.4 trend_snapshots
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| snapshot_date | DATE | 스냅샷 날짜 |
| keyword | VARCHAR | 키워드 |
| rank | INT | 해당 주 순위 |
| count | INT | 빈도 |
| rank_change | INT | 전주 대비 변동 (+/-) |
| is_new | BOOLEAN | 신규 진입 여부 |
